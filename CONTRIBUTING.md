# Contribution Guidelines

Thank you for your interest in contributing to Awesome Thymer!

## Adding a Resource

1. **Fork** this repository
2. **Add your resource** to the appropriate section in `README.md`
3. **Submit a pull request** with a clear description

## Requirements

- The resource must be related to Thymer
- The resource should be useful to the Thymer community
- Plugins should be functional and reasonably stable
- Include a brief, accurate description

## Formatting

Use this format:

```markdown
- [Resource Name](https://github.com/user/repo) - Brief description of what it does
```

- Keep descriptions concise (one line)
- Use sentence case for descriptions
- End descriptions without a period
- Place new entries alphabetically within their section

## Quality Standards

- No duplicate entries
- Links must be working
- No promotional or spam content
- Resources should be actively maintained (or clearly marked as archived)

## Categories

If your resource doesn't fit existing categories, feel free to suggest a new one in your PR description.

## Questions?

Open an issue if you have questions about whether a resource fits or where it should go.
